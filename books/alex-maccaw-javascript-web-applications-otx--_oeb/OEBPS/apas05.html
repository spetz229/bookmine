Being a Good Citizen

jQuery prides itself on being a good web citizen; as such, it is completely namespaced and doesn’t pollute global scope. However, the object jQuery is aliased to the $, which is often used by other libraries, such as Prototype. Therefore, to stop the libraries from conflicting, you need to use jQuery’s noConflict mode to change the alias and free up $:

var $J = jQuery.noConflict(); 

assertEqual( $, undefined );

When you’re writing jQuery extensions, you need to assume that jQuery’s no conflict mode has been switched on and that $ doesn’t reference jQuery. In practice, though, $ is a useful shortcut, so just make sure it’s a local variable:

(function($){

  // $ is a local variable      
  $(".foo").addClass("wem");

})(jQuery);

To simplify things, jQuery will also pass a reference to itself with the document.ready event:

jQuery(function($){
  // Runs when the page loads
  assertEqual( $, jQuery );
});
